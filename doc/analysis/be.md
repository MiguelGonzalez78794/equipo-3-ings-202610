# App Usability 
This document is meant to explain our app workflow. Is important to clarify that this document will only be the explanation of how to use our application, any question or concern regarding inner app design will be documented on a different correspondant file 

---

## Accessibility
For every instance in the app there will be at the top left corner a accesibility button wich can be used by the user at any moment, here any user will be able to use any basic feature to make their experience more comfortable. There are features such as invert colours, change font size, text to speech, etc.


## Landing Page
First, the user will go to the app's home page, where they will be welcomed and prompted to log in to our app via a button that will redirect them to the login screen

## Log In
Next, user will acces to the log in screen where they will be asked for their student credentials in order to log in via clicking 'Ingresar' button. There also will be alternatives for any user who has a log in problem, user will find a password recovery link in case they have forgot their password

## Password Recover
In case user has forgotten their password, whenever 'Recuperar Contraseña' on log in is clicked. User will be redirected to Password Recover page where they will be asked for its mail and a captcha test, where both of this are done user must press 'Recuperar' button for it to send they a mail with all the steps for password change

## Log Out
At any moment on the app, when the user wants to close it's session he will find a log out option on the menu where when it's pressed his session will expiere and user will be redirected to a log out page with no more instruction than to close current window

## Dashboard
After a succed log in, user will face app dashboard where they will find relevant info around the week. First, user will be welcomed, then they will be able to read about any news happening at the university then scrolling a bit down user will be able to face six quick access each of them functional to be used by pressing click over them

## Search
Whenever user acces Search interface they will be able to search any info relevant for them, they will encounter a search bar where is prompted the search and below it there will be a popular searches

## Menu
Whenever user has access to the app they will be able to acced the menu via a 'Menú' button at top right corner, menu will deploy and will display all features the app offers

## Map
User will be able to access the map, this is the main feature the app holds. Here any user can search any info about the campus, user will be guided via a map and they will be given a little piece of info about the place / activity they are searching about

## Profile
Every user has a profile,it could be acces via menu, where they will find out essential info like their ID, Mail, Exams, etc. A photo of the user will be displayed

## Mentors
User can access the mentors interface, via menu, where they cand search, ask and schedule consultations with the mentors university provides. Also mentor & user after scheduling something will create a private chat.

## Activities / Activities.Click
All users can access a activities interface, via menu, where they will find all activities happening now on campus, also user can click on any activitie for it to display all relevant information about it and also via 'Participar' button user can participate on that activitie
## Forum / Deployed Forum
All users can acces forum interface, via menu, where they will find piled questions made by other students, if user clicks on any question it will be deployed a more detailed version of the question where there is also other students responses 

## FAQ
Here a user can access FAQ interface, via menu, where they can send any FAQ to universitie internal servicies.
